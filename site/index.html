A real storage engine, in your pocket

Not a wrapper around SQLite — a from-scratch page-backed engine with modern internals.

🌐

Runs in the browser

Compiles to WebAssembly. Embed a real database in a web app — no backend required.

🕓

MVCC time-travel

Every write is append-only. Query any past state with SELECT … BEFORE tx.

💾

Larger than RAM

A bounded buffer pool streams pages on demand — a 667-page dataset serves from 16 pages.

🔗

Content-addressed dedup

Repeated blobs are stored once, keyed by BLAKE3 — up to 12× smaller on disk.

📊

Columnar compression

Cold pages transpose to Delta-Z + dictionary bit-packing — ~10× smaller.

🔍

Secondary indexes

Equality indexes turn WHERE col = v into a lookup — ~11,000× faster than a scan.

🧩

Sandboxed WASM UDFs

Run user-defined functions inside an isolated WebAssembly runtime.

🔒

Crash-safe durability

Opt-in Sync mode: fsync + atomic manifest commit (write-temp → rename).

Two lines from data to disk

Rust
use picovolt::Database;

let mut db = Database::open_dev("./app.pv")?;
db.query("CREATE TABLE users (id, name, tier)")?;
db.query("INSERT INTO users VALUES (1, 'alice', 'pro')")?;

// compile to a single, read-only, mmap'able file
db.bake("./app.pvdb")?;
JavaScript (npm + WebAssembly)
import init, { Db } from "picovolt";

await init();
const db = new Db();
db.query("CREATE TABLE users (id, name, tier)");
db.query("INSERT INTO users VALUES (1, 'alice', 'pro')");

const res = JSON.parse(db.query("SELECT * FROM users"));
